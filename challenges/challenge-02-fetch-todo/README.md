# Challenge 02 — Fetch Todo

Brief: Build a small Todo list that fetches initial items from a public API and practices side-effect data handling with `useEffect`, plus loading/error UX and simple local persistence.

Timebox: 1–3 hours

Primary focus: `useEffect` + browser `fetch` and handling loading/error states.

Secondary focus (optional): Robust state updates and simple persistence (`localStorage`).

Deliverables:

- Fetch initial todos on mount and render them.
- Show loading and error states with retry.
- Add new todos (controlled input), toggle completion, and show active/completed counts.
- Persist changes to `localStorage` so page reloads keep local edits.
